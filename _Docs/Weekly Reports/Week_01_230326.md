# Εβδομαδιαία Αναφορά 01 — Έως 23/03/26

## Επισκόπηση
Η πρώτη εβδομάδα του έργου επικεντρώθηκε στην αρχικοποίηση των υποδομών, την εκτενή έρευνα των διαθέσιμων τεχνολογιών και τον καθορισμό της αρχιτεκτονικής κατεύθυνσης της πτυχιακής εργασίας. Ο βασικός στόχος ήταν η δημιουργία ενός σταθερού πλαισίου πάνω στο οποίο θα βασιστεί η ανάπτυξη των επόμενων εβδομάδων.

## Ολοκληρωμένες Εργασίες
- **Διαμόρφωση Σκελετού Αναφοράς**: Πραγματοποιήθηκε ο σχεδιασμός της δομής της τελικής αναφοράς, περιλαμβάνοντας τον πίνακα περιεχομένων, τα απαραίτητα προσχέδια για την περίληψη και τη διάρθρωση των κεφαλαίων. Η δομή αυτή λειτουργεί ως οδικός χάρτης για τη συγγραφή της εργασίας παράλληλα με την ανάπτυξη του κώδικα.
- **Αρχικοποίηση Αποθετηρίου και Περιβάλλοντος Ανάπτυξης**: Δημιουργήθηκε το monorepo στο GitHub και ορίστηκαν οι ροές εργασίας (Git workflow). Επιλέχθηκε η δομή του monorepo για τη διασφάλιση της συνοχής μεταξύ backend, Android και Web εφαρμογών.
- **Έρευνα και Αξιολόγηση Τεχνολογιών**: Διεξήχθη ανάλυση των απαιτήσεων για multi-tenancy και cross-platform συμβατότητα. Μετά από αξιολόγηση, επιλέχθηκε το Kotlin Multiplatform (KMP) για το διαμοιρασμό της επιχειρησιακής λογικής, το Ktor για την ανάπτυξη του backend API και το Compose Multiplatform για τη δημιουργία των διεπαφών χρήστη (UI).
- **Συγγραφή Μέρους του 4ου Κεφαλαίου (Αρχιτεκτονική)**: Ξεκίνησε η τεκμηρίωση των σχεδιαστικών αποφάσεων στην ενότητα 4.5 "Αποφάσεις Σχεδίασης και Επιλογή Τεχνολογιών", όπου αναλύεται το σκεπτικό πίσω από την επιλογή του συγκεκριμένου tech stack σε σχέση με τις ανάγκες του συστήματος.

## Προγραμματισμός Επόμενης Περιόδου
Για την επόμενη εβδομάδα προγραμματίζεται η ολοκλήρωση της ανάλυσης απαιτήσεων (Κεφάλαιο 3) και η έναρξη του σχεδιασμού του μοντέλου δεδομένων, καθώς και η υλοποίηση των πρώτων οντοτήτων της βάσης δεδομένων.
